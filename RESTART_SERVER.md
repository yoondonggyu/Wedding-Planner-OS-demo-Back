# 백엔드 서버 재시작 필요

## 문제 상황
- 데이터베이스 컬럼은 모두 존재함 ✅
- SQLAlchemy 모델도 올바르게 정의됨 ✅
- 하지만 여전히 "Unknown column 'groom_father_name'" 오류 발생

## 원인
SQLAlchemy가 서버 시작 시 데이터베이스 스키마를 캐시했기 때문입니다.
데이터베이스에 컬럼을 추가한 후 서버를 재시작하지 않으면 이 오류가 발생합니다.

## 해결 방법

### 방법 1: 백엔드 서버 재시작 (권장)
1. 현재 실행 중인 백엔드 서버를 중지 (Ctrl+C)
2. 백엔드 서버를 다시 시작:
   ```bash
   cd 2.Wedding_OS_back
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```

### 방법 2: 서버가 자동 재시작되도록 설정
`--reload` 옵션이 활성화되어 있으면 코드 변경 시 자동으로 재시작됩니다.
하지만 데이터베이스 스키마 변경은 코드 변경이 아니므로 수동 재시작이 필요합니다.

## 확인 사항
재시작 후 다음을 확인하세요:
- ✅ 모든 컬럼이 존재하는지 확인됨
- ✅ 모델에 모든 컬럼이 정의되어 있음
- ✅ 재시작 후 정상 작동할 것입니다
