# HTTP 내용 정리

## 기본 개념
- **HTTP (HyperText Transfer Protocol)**: 인터넷 상에서 클라이언트와 서버가 데이터를 주고받기 위한 약속된 통신 규약이다.
- **Request (요청)**: 클라이언트가 서버에게 원하는 동작이나 자원을 달라고 보내는 메시지이다.
- **Response (응답)**: 서버가 클라이언트의 요청을 처리하고 그 결과를 담아 보내는 메시지이다.
- **URL (Uniform Resource Locator)**: 인터넷 상에 있는 자원(파일, 페이지 등)의 위치를 나타내는 주소이다.

## HTTP 메서드 (Method)
- **GET**: 서버로부터 데이터를 조회하거나 가져올 때 사용하는 요청 방식이다.
- **POST**: 서버에 새로운 데이터를 생성하거나 작업을 처리해달라고 요청할 때 사용한다.
- **PUT**: 리소스 전체를 수정하거나, 없으면 생성할 때 사용하는 방식이다 (덮어쓰기 느낌).
- **PATCH**: 리소스의 일부분만 수정할 때 사용하는 방식이다.
- **DELETE**: 서버에 있는 특정 리소스를 삭제해달라고 요청할 때 사용한다.

## 상태 코드 (Status Code)
- **200 OK**: 요청이 성공적으로 처리되었음을 의미한다.
- **201 Created**: 요청이 성공해서 새로운 리소스가 생성되었음을 의미한다.
- **400 Bad Request**: 클라이언트가 잘못된 요청을 보내서 서버가 처리할 수 없음을 의미한다.
- **401 Unauthorized**: 인증이 필요한 요청인데 인증되지 않았음을 의미한다 (로그인 필요).
- **403 Forbidden**: 서버가 요청을 이해했지만, 권한이 없어서 거절함을 의미한다.
- **404 Not Found**: 요청한 리소스를 찾을 수 없음을 의미한다.
- **500 Internal Server Error**: 서버 내부에서 문제가 발생하여 요청을 처리하지 못했음을 의미한다.

## 기타 특징
- **Header (헤더)**: 요청이나 응답에 대한 부가적인 정보(메타데이터)를 담는 영역이다.
- **Body (바디)**: 실제 전송하려는 데이터(JSON, HTML, 파일 등)를 담는 영역이다.
- **Stateless (무상태성)**: 서버가 클라이언트의 이전 상태를 기억하지 않고, 매 요청을 독립적으로 처리하는 특성이다.
- **Connectionless (비연결성)**: 요청에 대한 응답을 보내면 연결을 끊어 서버 자원을 효율적으로 쓰는 특성이다.
